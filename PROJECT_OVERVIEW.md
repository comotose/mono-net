# MONO — объяснение проекта

## Что это за проект

`MONO` — это социальная веб-платформа с базовыми возможностями:

- регистрация и вход пользователей;
- личный профиль;
- лента публикаций;
- лайки и комментарии;
- подписки на пользователей;
- поиск людей;
- личные сообщения (текст, файлы, изображения, голосовые).

Проект написан на Laravel и использует серверный рендеринг Blade-шаблонов, а интерактивные части (лайки, realtime-чат) дополняются JavaScript.

---

## Как работает приложение (по шагам)

1. Пользователь открывает URL.
2. Запрос попадает в `routes/web.php`.
3. Маршрут вызывает нужный контроллер (`app/Http/Controllers/...`).
4. Контроллер получает/изменяет данные через модели Eloquent (`app/Models/...`).
5. Данные сохраняются/читаются из MySQL.
6. Контроллер возвращает Blade-представление (`resources/views/...`).
7. На клиенте подключаются стили и скрипты через Vite.
8. Для realtime-сообщений используется Echo + Pusher (события из Laravel).

## Что делает каждая технология

### Laravel

Основной backend-фреймворк проекта. Отвечает за:

- маршрутизацию;
- контроллеры и бизнес-логику;
- валидацию форм;
- работу с БД через Eloquent ORM;
- миграции структуры БД;
- аутентификацию и авторизацию;
- отправку broadcast-событий в realtime-чат.

### PHP

Язык, на котором написан backend проекта (контроллеры, модели, события, middleware).

### MySQL

Основная база данных. Хранит пользователей, посты, комментарии, лайки, сообщения, подписки и другие сущности.

### Blade

Шаблонизатор Laravel для генерации HTML на сервере. Используется для страниц:

- лента;
- профиль;
- поиск;
- чат;
- формы авторизации.

### Tailwind CSS

Утилитарный CSS-фреймворк для стилизации интерфейса. Позволяет быстро собирать дизайн прямо в классах HTML.

### Alpine.js

Легкий JavaScript-фреймворк для небольшой реактивности на страницах Blade (переключатели, раскрывающиеся блоки и простое UI-поведение).

### Vite

Сборщик фронтенда. Отвечает за:

- dev-сервер в разработке (`npm run dev`);
- сборку статики для production (`npm run build`);
- генерацию `public/build/manifest.json` для подключения ассетов в Blade.

### Laravel Echo

Клиентская библиотека для подписки на broadcast-события Laravel в браузере. Используется в чате, чтобы получать новые сообщения без перезагрузки.

### Pusher (pusher-js + pusher-php-server)

Транспорт realtime-сообщений:

- `pusher-php-server` на backend отправляет события;
- `pusher-js` на frontend получает события через websocket.

### Laravel Breeze

Готовый стартовый набор аутентификации:

- регистрация;
- логин/логаут;
- восстановление пароля;
- подтверждение email.

## Где какая логика в проекте

- `routes/web.php` — все веб-маршруты.
- `app/Http/Controllers` — логика HTTP-действий.
- `app/Models` — модели и связи БД.
- `app/Events` — события (например, отправка сообщений в чат).
- `database/migrations` — версия структуры базы данных.
- `resources/views` — Blade-шаблоны интерфейса.
- `resources/js/app.js` — общий frontend-код.
- `resources/js/echo-chat.js` — realtime-логика чата.
- `resources/css/app.css` — стили проекта.

## Как запускается проект

1. Установить PHP/Composer/Node.js/MySQL.
2. Установить зависимости:
   - `composer install`
   - `npm install`
3. Настроить `.env` (БД, APP_URL, broadcast).
4. Выполнить:
   - `php artisan key:generate`
   - `php artisan migrate`
   - `php artisan storage:link`
5. Запустить:
   - `php artisan serve`
   - `npm run dev`

## Какой подход у интерфейса

Сейчас проект — это в основном **SSR + интерактивные участки**:

- страницы рендерятся на сервере через Blade;
- отдельные действия работают без полного reload (например лайки/realtime чат);
- при необходимости проект можно развивать в сторону более SPA-поведения (через fetch/Inertia/Livewire).

---

## Кратко: зачем этот стек

- Laravel + Blade — быстро и стабильно для CRUD и соц-функций.
- Tailwind + Alpine — простой и легкий frontend без перегруза.
- Echo + Pusher — realtime там, где он действительно нужен (чат).
- Vite — современная и быстрая сборка ассетов.
